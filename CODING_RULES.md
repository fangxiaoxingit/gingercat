# GingerCat 编码规则

## 1. 注释规则（强制）

1. 每次提交代码时，所有新增或修改的逻辑都必须补充注释。
2. 注释要说明“为什么这样做”，不要只重复代码字面含义。
3. 至少在以下位置添加注释：
   - 新增方法/函数前
   - 复杂条件分支前
   - 异步流程与错误兜底逻辑前
   - 数据模型新增字段处（说明用途）
4. 注释语言统一使用中文。
5. 禁止提交无意义注释（例如“给变量赋值”）。

## 2. 注释示例

```swift
// 先走本地 OCR，确保无论 AI 开关状态都能获得基础文本。
let recognition = try await VisionOCRService.recognize(from: image)

// AI 开启时走一次性结构化提取，失败则回退到本地摘要，保证数据结构一致。
if aiSummaryEnabled, config.canRequestSummary {
    ...
}
```

## 3. 提交前自检（强制）

1. 检查本次改动是否都包含必要注释。
2. 检查注释是否准确描述业务意图与边界。
3. 确保注释与实际代码一致，避免过期注释。
