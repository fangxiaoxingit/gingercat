# 大橘小事 (GingerCat miao) iOS 26 全案开发需求文档 (PRD)

## 1. 产品定义与核心价值
- **名称**：GingerCat miao
- **暂定中文名**：大橘有事
- **核心功能**：提取图片/截图文本 → AI 意图分类与总结 → 一键智能录入 iOS 日历或待提醒事项
- **设计哲学**：Liquid Glass (液态玻璃) —— 结合流体动力学与半透明材质，强调信息在玻璃表面"折射"与"流动"的未来感

## 2. 核心功能需求
1. **扫描功能**：打开应用显示历史记录列表，右下浮动拍照按钮，点击后调用全屏拍照功能，右下角可选择相册上传
2. **OCR 识别**：获取图片后，调用 iOS 原生 API 获取图片中的文本信息及坐标信息
3. **AI 处理**：处理文本信息及坐标信息，总结文字信息，判断是否有日程信息（带日期相关的事件）
4. **日程创建**：如果有完整的事件内容和待提醒日期，提醒用户是否创建日程
5. **历史记录管理**：历史记录列表、历史记录详情、删除、编辑、备注、截图分享、加入提醒事项
6. **设置功能**：支持自定义 baseURL、API key 设置

## 3. 视觉与设计规范 (Liquid Glass System)
### 3.1 材质与层级
- **容器**：统一使用 .ultraThinMaterial，搭配 containerRelativeShape 保证圆角与系统物理边缘对齐
- **边缘**：0.5pt 的白色半透明描边（.white.opacity(0.3)），模拟玻璃切角
- **背景**：使用动态 MeshGradient（颜色：靛青、深紫、海蓝），通过 PhaseAnimator 实现非线性流动感
- **图标**：仅使用 SF Symbols（如 shimmer.circle.fill, sparkles.rectangle.stack），优先开启多色模式

### 3.2 颜色系统
#### 3.2.1 主题色
- **主色调**：#346739（森林绿色）
  - 用途：主要操作按钮、重要信息高亮、品牌标识
  - 色彩心理：稳定、自然、可信赖、成长感

#### 3.2.2 辅助配色
- **深色变体**：
  - #28522D（深绿色）- 用于按下状态、次要操作
  - #1F4324（深墨绿）- 用于边框、分割线
  
- **浅色变体**：
  - #4D8552（浅绿色）- 用于悬停状态、背景装饰
  - #D7E6D8（淡绿色）- 用于轻微强调、背景渐变
  
- **互补色**：
  - #5A6AF2（蓝紫色）- 用于对比元素、信息提示
  - #6A5AF2（紫色）- 用于特殊状态、品牌辅助
  
- **中性色**：
  - #FFFFFF（白色）- 主要背景、文字
  - #F8F8F8（浅灰）- 次要背景、卡片
  - #E0E0E0（中灰）- 分割线、边框
  - #666666（深灰）- 次要文字
  - #333333（黑灰）- 主要文字
  
- **功能色**：
  - #4CAF50（绿色）- 成功状态、确认操作
  - #F44336（红色）- 错误状态、删除操作
  - #2196F3（蓝色）- 信息提示、链接

#### 3.2.3 渐变配色
- **主渐变**：#346739 → #4D8552（绿色渐变）
- **背景渐变**：靛青 → 深紫 → 海蓝（液态玻璃背景）
- **功能渐变**：#346739 → #6A5AF2（主题到互补色）

### 3.3 交互反馈
- **触感 (Haptics)**：扫描时使用 .soft 模拟液体沸腾；成功时使用 .success
- **主题**：深浅模式自动适配。深色模式需增加微弱的蓝色荧光（Cyan Glow）

### 3.4 液态玻璃设计约束条件
- **系统框架利用**：优先使用系统标准组件（SwiftUI、UIKit），让系统自动处理液态玻璃效果
- **自定义背景限制**：减少控制和导航元素中自定义背景的使用，避免覆盖或干扰液态玻璃效果
- **辅助功能适配**：确保应用在不同显示和辅助功能设置下正常运行，特别是减少透明度和运动的设置
- **效果使用克制**：避免过度使用液态玻璃效果，只在最重要的功能元素上应用
- **图标设计规范**：
  - 提供视觉一致、光学平衡的设计
  - 考虑简化的设计，由实心、填充、重叠的半透明形状组成
  - 让系统处理遮罩、模糊和其他视觉效果
  - 使用 Icon Composer 预览和组合图标
- **系统适配**：确保应用在不同设备和系统版本上的一致性，特别是液态玻璃效果的表现
- **性能考虑**：液态玻璃效果可能影响性能，需在实现中注意优化

## 4. 页面功能模块说明
### 4.1 主页 (Home / Scanner)
- **视觉**：极简界面。中心为一个"液态感"十足的 Glass Orb 交互球
- **动作**：点击呼起照片选择器；长按启动相机；支持系统级文件拖拽进入

### 4.2 结果分析页 (Insight Drawer)
- **AI 总结**：顶部显示由 AI 生成的 50 字以内内容摘要
- **意图判定 (Gatekeeper)**：
  - **日程模式**：识别到时间信息，显示可勾选的事件列表，支持处理多个时间点（如上映日 vs 抢票日）
  - **总结模式**：若判定为非日程图片（如风景照），仅显示总结，隐藏"写入日历"按钮

### 4.3 历史记录 (Archive)
- **数据**：基于 SwiftData 本地存储
- **搜索**：模糊搜索 AI 生成的总结词
- **交互**：侧滑删除动画表现为"液滴消散"效果

### 4.4 设置与自定义模型 (Neural Engine)
- **多语言**：UI 字符串全部使用 String(localized:)
- **模型配置**：用户可选择系统原生模型或自定义 API Key（必须存储在 Keychain 中）、Base URL 和 Model ID

## 5. iOS 桌面小组件 (WidgetKit)
- **设计要求**：背景使用静态 MeshGradient + 玻璃材质，通过 App Group 与主应用共享数据

## 6. 技术架构与 API 约束
### 6.1 技术栈要求
- **主要语言**：Swift 6.0+
- **UI 框架**：SwiftUI 5.0+，充分利用系统内置组件实现液态玻璃效果
- **数据存储**：SwiftData 用于本地数据持久化
- **OCR 识别**：使用 Vision 框架进行图片文本识别
- **AI 集成**：通过 API 调用实现文本分析和意图识别
- **安全存储**：Keychain 用于存储敏感信息（如 API Key）
- **小组件**：WidgetKit 实现 iOS 桌面小组件

### 6.2 技术实现约束
- **系统要求**：iOS 26.0+
- **Xcode**：最新版本 Xcode，确保支持液态玻璃效果
- **架构模式**：采用 MVVM 或 VIPER 架构，确保代码可维护性
- **性能优化**：
  - 图片处理和 OCR 识别在后台线程执行
  - AI 调用采用异步处理，避免阻塞主线程
  - 液态玻璃效果的实现需考虑性能影响
- **隐私保护**：
  - 敏感数据（如 API Key）必须存储在 Keychain 中
  - 图片处理优先在本地进行，减少网络传输
  - 遵循苹果隐私政策和权限要求

## 7. AI 协作开发流程 (Roadmap)
1. **第一阶段（视觉）**：编写 LiquidBackground.swift 和 GlassCard.swift 基础组件
2. **第二阶段（核心）**：封装 ImageProcessor 模块（Vision OCR + API 调用逻辑）
3. **第三阶段（系统）**：实现 CalendarManager（日历写入）与 KeychainHelper（密钥加密）
4. **第四阶段（外延）**：开发 WidgetKit 扩展，实现桌面方形与长形列表组件
5. **第五阶段（整合）**：接入全局震动反馈、多语言支持及深浅模式调优
